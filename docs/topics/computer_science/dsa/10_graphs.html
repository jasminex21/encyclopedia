Computer Science
Published

Introduction

A graph is a non-linear data structure consisting of a set of nodes or vertices and edges that connect the nodes. Edges can be directed or undirected, and can optionally have values (weighted graphs).

Trees are examples of undirected graphs in which any two nodes are connected by exactly one edge, and there can be no cycles in the graph.

Graphs are commonly used to model relationships between unordered entities, such as friendships between people, or distances between locations.

It is useful to be familiar with the various graph representations, graph search algorithms, and their time and space complexities.

Graph Representations

In order to traverse a graph, you must first have a representation of the graph. The most common graph representations are:

  • Adjacency matrix
  • Adjacency list
  • Hash table of hash tables

Breadth-First Search (BFS)

Breadth-first search is a graph traversal algorithm which starts at a node and explores all nodes at the current depth before mocing on to the nodes at the next depth level.

Typically, a queue is used to keep track of the nodes that have been visited but not yet explored.

Queues in Python should be implemented via the deque module, which allows us to use double-ended queues (\(O(1)\) dequeuing) and not Python lists (\(O(n) dequeuing\)).

from collections import deque

def bfs(matrix):

    if not matrix: 
        return []

    rows, cols = len(matrix), len(matrix[0])
    visited = set()

    # this is iterative; it runs until the queue is empty
    def traverse(r, c): 

        queue = dequeue()
        # mark the position as visited
        visited.add((r, c))
        # add the position to the queue
        queue.append((r, c))

        # while queue is not empty
        while queue: 
            # pop from queue
            row, col = queue.popleft()
            # check adjacent positions from the popped positoin
            directions = [[1, 0], [-1, 0], [0, 1], [0, -1]]

            for dr, dc in directions: 
                # check if position is in bounds
                # and that the position has not already been visited
                if ((r + dr) in range(rows)) and ((c + dc) in range(cols)) and ((r + dr, c + dc) not in visited): 
                    # add to queue to run BFS later
                    queue.append((r + dr, c + dc))
                    # mark as visited
                    visited.add((r + dr, c + dc))

    visited = set()
    directions = ((0, 1), (0, -1), (1, 0), (-1, 0))

    def traverse(i, j): 
        queue = deque([(i, j)])

        while queue: 
            # first in, first out
            curr_i, curr_j = queue.popleft()
            if (curr_i, curr_j) not in visited: 
                visited.add((curr_i, curr_j))
                # traverse all neighbors
                for direction in directions: 
                    next_i, next_j = curr_i + direction[0], curr_j + direction[1]

                    if 0 <= next_i < rows and 0 <= next_j < cols:

                        queue.append((next_i, next_j))

    for i in range(rows): 
        for j in range(cols): 
            traverse(i, j)

Dijkstra’s Algorithm

Dijkstra’s algorithm is commonly used to solve single-source shortest-path problems (with non-zero edge weights). Dijkstra’s algorithm finds the shortest paths from a single source node to all other nodes in a graph with non-negative edge weights. This is usually implemented using a priority queue (i.e., a min-heap) for efficiency.

The algorithm can be summarized as follows:

  1. Start from the source, which has distance (weight) 0
  2. Always expand the currently known closest unvisited node (greedy choice). I.e., the next node you explore should be the one linked to the previous node with the smallest edge weight.
  3. Once a node is expanded, its shortest distance is finalized and never updated again.
  4. Use a priority queue (min-heap) to always pick the next closest node.

General implementation:

        w1, n1 = heapq.heappop(min_heap)

        if n1 in visited: 
            continue
